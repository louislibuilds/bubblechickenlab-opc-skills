# OPC Founder OS

Operating system for **you** — the solo founder. Not the product.

## When to Activate

Triggers: weekly plan, prioritize, focus, burnout, what should I work on, too many tasks, WIP, context switch.

**Out of scope:** technical implementation (→ opc-build-engine), detailed product spec (→ opc-product-thinking).

## Weekly Loop

```
Monday: Pick 1 primary outcome + 2 supporting tasks
Daily: One deep-work block (2–4h) on primary
Friday: Ship note or honest retrospective (15 min)
```

## Priority Stack

Rank backlog with **ICE** (solo version):

| Factor | Question |
|--------|----------|
| Impact | Moves north star or unblocks revenue? |
| Confidence | Do I know how to ship it this week? |
| Effort | ≤2 days solo? |

Do top item only until shipped or explicitly killed.

## WIP Limits

- **1** active product initiative
- **1** active marketing push (can be same Ticket)
- Park everything else in `Backlog.md` or issue list — don't context-switch daily

## Checklist

- [ ] One primary outcome named for the week
- [ ] Max 3 committed tasks
- [ ] Explicit "not this week" list
- [ ] Recovery block if last week failed goals
- [ ] Link tasks to @opc-os Tickets where possible

## Burnout Guards

- No guilt-driven scope expansion
- If tired: shrink MVP, don't abandon PLAN MODE
- Ship small > perfect plan
- Social/content optional when primary engineering slips

## Advisory Format

```markdown
### opc-founder-os
- [SUGGESTION] This week: landing SEO only — defer theme sprint
- [SUGGESTION] Block 9–12 Tue/Thu for implementation; content Friday PM
```
